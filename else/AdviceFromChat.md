# 六、你真正需要的“RAG能力”

很多人会误以为：

> “调用LLM API”就是AI项目

不是。

真正核心：

# “检索工程”

你需要重点研究：

***

## 1. Chunk切分策略（最重要）

这是RAG命脉。

推荐：

```
chunk_size = 300~500 中文字
overlap = 50~80

```

并：

# 按语义切分

不要硬切。

***

## 2. Metadata设计

每个chunk必须保存：

```
{
  "source": "...",
  "title": "...",
  "department": "...",
  "url": "...",
  "updated_at": "...",
  "doc_type": "faq/pdf/notice"
}

```

后期非常重要。

***

## 3. 引用系统

必须支持：

- 来源链接
- 来源标题
- 点击跳转

这是：

# “可信AI”

的核心。
